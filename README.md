Roadmap
Stage 1 — Core trip organizer (current)

 Trip CRUD with automatic status detection
 Airline auto-detection from booking reference
 Hotel AI parsing via Claude API
 GPS departure location detection
 Hotel gap notification
 Business trip receipt tracking
 Persist data with AsyncStorage
 Navigation between screens (React Navigation)
 App Store submission via TestFlight

Stage 2 — Social + ML personalization

 User authentication
 ML-powered restaurant and activity recommendations based on past trips
 Shared trips — collaborate on an itinerary with travel companions
 Book and manage trips on behalf of others (e.g. parents)

Stage 3 — GenAI trip generation

 Natural language trip planning: "5 days in Hawaii, beach, relax"
 Budget and group size constraints
 MBTI-aware planning 
 Full destination recommendations from scratch


About This Project
TripPlanner is a personal project built to:

Solve a real travel pain point experienced firsthand
Build full-stack mobile development skills as a transition from QA/ML testing into a developer/MLE role
Demonstrate end-to-end product ownership — from problem definition to App Store delivery

The project intentionally progresses from a simple organizer (Stage 1) to ML personalization (Stage 2) to GenAI generation (Stage 3), mirroring a real product roadmap.
